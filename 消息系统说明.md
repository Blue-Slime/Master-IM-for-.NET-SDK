# MasterIM 消息系统说明

## 1. Message 字段说明

```csharp
public class Message
{
    public int PageNumber { get; set; }        // 页号（客户端设置为0）
    public int InPageSeq { get; set; }         // 页内序号（客户端设置为0）
    public string Content { get; set; }        // 消息内容（支持任意格式）
    public DateTime SendTime { get; set; }     // 发送时间
    public string SenderId { get; set; }       // 发送者ID
    public DateTime? ReplyToTime { get; set; } // 回复的消息时间（可选）
    public string? QuotedContent { get; set; } // 引用内容（可选）
    public string? RoleId { get; set; }        // 角色ID（可选）
}
```

## 2. 页号和序号分配

**当前实现**：客户端发送时设置为 `0`，服务端**直接存储**客户端传来的值。

**问题**：所有消息的 PageNumber 和 InPageSeq 都是 0，无法正确分页。

**需要修复**：服务端应自动分配页号和序号。


## 3. 自定义消息类型

**当前支持**：`Content` 字段是 `string` 类型，支持任意内容。

**实现方式**：
```csharp
// 方式1：JSON格式
await client.SendMessageAsync(new Message
{
    Content = "{\"type\":\"image\",\"url\":\"https://...\"}",
    SenderId = "user1",
    SendTime = DateTime.Now
});

// 方式2：自定义协议
await client.SendMessageAsync(new Message
{
    Content = "[IMG]https://...[/IMG]",
    SenderId = "user1",
    SendTime = DateTime.Now
});
```

**建议**：添加 `MessageType` 字段区分消息类型。


## 4. 总结

### 当前状态
✅ **已支持**：
- 基础文本消息
- 消息回复/引用
- 角色扮演（RoleId）
- 任意格式内容（通过Content字段）

❌ **需要改进**：
- 页号和序号未自动分配
- 缺少MessageType字段
- Content字段未结构化

### 建议改进
1. 服务端自动分配PageNumber和InPageSeq
2. 添加MessageType字段（text, image, audio, video, file, custom）
3. 添加MessageData字段存储结构化数据
